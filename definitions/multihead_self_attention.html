Multi-Head Self-Attention

Multi-head self-attention is a key mechanism in transformer models like GPT. It allows the model to focus on different parts of the input sequence simultaneously, capturing various relationships and patterns in the data.

  • Each "head" learns to attend to different positions in the sequence.
  • Helps the model understand context, dependencies, and meaning across the input.
  • Outputs from all heads are combined to form a richer representation.
Figure: Each attention head processes the input sequence independently, then their outputs are concatenated and projected to form the final representation.

How It Works (Step-by-Step)

  1. Input Embeddings: Each token in the sequence is converted to a vector.
  2. Linear Projections: For each head, the input is projected into queries (Q), keys (K), and values (V).
  3. Attention Scores: Each head computes attention scores using Q and K, then applies them to V.
  4. Concatenation: Outputs from all heads are concatenated and linearly transformed.

Code Example from microgpt.py

Below is a simplified excerpt from our gpt() function, showing how multi-head self-attention is implemented:

for li in range(n_layer):
    # Multi-head Attention block
    x_residual = x
    x = rmsnorm(x)
    q = linear(x, state_dict[f'layer{li}.attn_wq'])
    k = linear(x, state_dict[f'layer{li}.attn_wk'])
    v = linear(x, state_dict[f'layer{li}.attn_wv'])
    keys[li].append(k)
    values[li].append(v)
    x_attn = []
    for h in range(n_head):
        hs = h * head_dim
        q_h = q[hs:hs+head_dim]
        k_h = [ki[hs:hs+head_dim] for ki in keys[li]]
        v_h = [vi[hs:hs+head_dim] for vi in values[li]]
        attn_logits = [sum(q_h[j] * k_h[t][j] for j in range(head_dim)) / head_dim**0.5 for t in range(len(k_h))]
        attn_weights = softmax(attn_logits)
        head_out = [sum(attn_weights[t] * v_h[t][j] for t in range(len(v_h))) for j in range(head_dim)]
        x_attn.extend(head_out)
    x = linear(x_attn, state_dict[f'layer{li}.attn_wo'])
    x = [a + b for a, b in zip(x, x_residual)]
Each head processes a different "view" of the input, and their outputs are combined for a richer understanding of context.

Why Multi-Head Self-Attention Matters

  • Allows the model to capture complex relationships in the data.
  • Improves the ability to model long-range dependencies and context.
  • Is a foundational building block for modern transformer architectures like GPT, BERT, and more.
